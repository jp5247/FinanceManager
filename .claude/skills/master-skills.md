# Master Skills

Project-wide baseline standards for all agents.

## Engineering standards

- Prefer small, reversible changes.
- Keep public APIs stable unless changes are explicitly requested.
- Add tests for behavior changes.
- Preserve existing coding style within each file.

## Security standards

- Never log secrets, credentials, or private keys.
- Validate untrusted input at boundaries.
- Prefer least-privilege defaults.

## Delivery standards

- Explain tradeoffs and assumptions.
- Call out risks and mitigations.
- Include validation steps in plans.

## Cost standards

- Favor pay-as-you-go designs with explicit cost controls.
- Include observability needed for cost monitoring.
